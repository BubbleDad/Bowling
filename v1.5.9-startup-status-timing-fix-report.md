# Meowmoon Bowling v1.5.9 Startup Status Timing Fix Report

## Purpose

This release fixes the reproducible, recoverable first-frame rotating-status error without changing gameplay or animations.

## Root cause

The first `requestAnimationFrame` timestamp can represent the start of a browser frame that began just before `startLevel()` assigned `game.titleStartedAt` using `performance.now()`. The resulting small negative elapsed value was floored to `-1`. JavaScript preserves the sign for `%`, so the status-message array was indexed with `-1`, returning `undefined`; `drawWrappedStatusText()` then failed at `text.split(" ")`.

## Change

- Clamp rotating-status elapsed time with `Math.max(0, current - game.titleStartedAt)`.
- Retain the existing rotation calculation for all non-negative elapsed times.
- Fall back to `visibleTexts[0]` if a status lookup is ever unavailable.

## Intentionally not changed

- No gameplay, scoring, animations, pin counts, startup-overlay behavior, phase-state behavior, lifecycle logic, ball specials, or visual design.
- No service-worker behavior beyond the required version and cache-name update.

## Version/cache

- App file: `app-v1.5.9.js`
- Cache name: `meowmoon-bowling-v1-5-9-startup-status-timing-fix-cache`
