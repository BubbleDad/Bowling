# v1.5.17 Cardinal Fly-Off Top Safe-Area Report

## Summary

Fixes `cardinalflyoff` / Cardinal Flies Off Screen v1 clipping when launched from top-row pins.

## What changed

- Added a top-edge safe-area offset in `cardinalflyoffv1DrawPin`.
- Mirrored the same wrapper fix in the gallery preview runtime.
- The draw function, registry key, duration, bird artwork, wing motion, and final offscreen exit behavior are unchanged.

## Safe-area estimate

The launch frame places the bird center at `anchorY - 126 * scale`.

The visible top extent of the launch pose, including head, crest/beak area, and the raised wing envelope, is estimated at about `48 * scale` above that center.

Total launch top extent used:

```text
174 * scale
```

The wrapper uses an 8 px top margin:

```js
const safeYOffset = Math.max(0, 8 - (baseAnchorY - 174 * scale));
```

This shifts the launch pose down only when its estimated top would otherwise be above the visible canvas.

## Validation

- Top-row static check confirms the launch top is moved to the 8 px margin.
- Middle-row static check confirms no offset is applied when there is enough top room.
- Bottom-row static check confirms no offset is applied and the fix does not add bottom clipping pressure.
- JavaScript syntax checks passed for the production app and gallery runtime.
