# Meowmoon Bowling v1.4.5 Rendering and Code-Section Cleanup

## Purpose

This is cleanup implementation 4. It improves code navigation and rendering organization without changing gameplay, animation artwork, pin counts, or rules.

## Main change

The old `render(current)` function directly called every drawing function in one long sequence. It now delegates to named rendering layers:

- `drawPlayfieldLayer(current)`
- `drawPinAndParticleLayer(current)`
- `drawPlayerAndBallLayer(current)`
- `drawInterfaceLayer(current)`
- `drawOverlayLayer(current)`

The visual stacking order is intended to remain the same as v1.4.4.

## Section headers added

Large section headers were added for:

- code map
- rendering pipeline
- playfield and background drawing
- pin drawing
- special pin animation drawing
- player, ball, and main UI drawing
- particle, star, heart, and paw drawing helpers
- main loop and startup

## Maintenance guidance

When revising rendering code:

1. Do not change the layer order unless you intend to change visual stacking.
2. Keep state changes out of drawing functions.
3. Drawing functions should draw only.
4. Pin state changes belong in lifecycle/update functions.
5. New animation drawing functions should be registered in `SPECIAL_ANIMATION_REGISTRY`.

## What did not change

- No new animations.
- No animation art revisions.
- No gameplay changes.
- No phase-state changes.
- No pin count changes.
- No cache behavior changes except the normal version/cache bump.
