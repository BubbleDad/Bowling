# Meowmoon Bowling v1.4.2 Cleanup Registry Refactor Report

## Purpose

This release is intended to be a behavior-preserving cleanup pass. It does not add new animations, revise animation artwork, change pin counts, or change gameplay goals.

## Main structural change

Special pin animation metadata has been centralized in `SPECIAL_ANIMATION_REGISTRY`.

The registry now holds, for each special pin animation:

- `label`
- `duration`
- `updateMode`
- `overlay`
- `exclusiveGroup`
- `draw`

## Behavior intended to remain the same

The following behavior is intended to match v1.4.1:

- 12 to 16 pins per level.
- Dandelion, heart, and sun animations draw above pins.
- Only one sun animation is selected per level.
- Existing animation durations are preserved.
- Existing animation drawing functions are preserved.
- Existing update behavior is preserved by `updateMode`.

## What changed internally

The code now uses registry helper functions:

- `animationSpec(type)`
- `animationDuration(type)`
- `animationUpdateMode(type)`
- `isOverlayAnimation(type)`
- `animationDrawFunction(type)`
- `buildPinSpecialQueue()`
- `validateSpecialAnimationRegistry()`

## Where to add or remove animations

Use the registry first. Do not add a new animation only to a drawing function.

To add a special pin animation:

1. Add a new entry in `SPECIAL_ANIMATION_REGISTRY`.
2. Set a duration function.
3. Set the correct `updateMode`.
4. Set `overlay: true` only if the animation must draw above pins.
5. Set `exclusiveGroup` only if only one animation from that group may appear per level.
6. Add or reference the draw function.
7. Test in the animation workshop before game integration.

## Important cache/deployment note

This release uses:

- `app-v1.4.2.js`
- `meowmoon-bowling-v1-4-2-cleanup-registry-cache`

The versioned filename is intentional. It reduces the risk that an old cached `app.js` or old versioned file is loaded during testing.
