# Meowmoon Bowling v1.4.8 Start Overlay Fix

## Purpose

This release addresses the persistent intermittent startup issue where the canvas title screen sometimes does not appear and the game will not start until multiple refreshes.

## What changed

- Added a DOM-based `#startOverlay` that displays "Tap anywhere to roll" independently of canvas rendering.
- Added a pointer handler to the start overlay.
- Added `showStartOverlay()` and `hideStartOverlay()`.
- The title/start path can now be triggered from either the canvas or the overlay.
- The main animation loop now uses `try/finally` so a render/update error cannot permanently stop future animation frames.
- Added an emergency canvas start screen fallback if the render loop catches an error.

## Why this is different from v1.4.7

v1.4.7 tried to repair startup state inside the game. v1.4.8 also provides a separate HTML start layer, so the user-facing start prompt is not dependent on the canvas title render succeeding.

## What did not change

- No new animations.
- No animation art changes.
- No gameplay rule changes.
- No pin count changes.
- No special animation frequency changes.
