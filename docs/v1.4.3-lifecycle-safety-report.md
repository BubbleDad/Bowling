# Meowmoon Bowling v1.4.3 Lifecycle Safety Pass

## Purpose

This is the second cleanup implementation. It is intended to preserve normal gameplay behavior while adding defensive safeguards around pin lifecycle, special animation lifecycle, ball rolling, and level resolving.

## What changed

### Pin lifecycle guardrails

Added helper functions:

- `markPinRemoved(pin, reason)`
- `activePinCount()`
- `repairPinLifecycle(pin, current)`
- `clearLingeringResolvedPins()`

These are used to safely remove or advance pins that are already in a broken or overlong lifecycle state.

### Special animation guardrails

Special animation updates are now wrapped in `try/catch`. If a special animation update throws, the affected pin is removed safely so the game does not freeze.

Special animations also have a grace timeout after their expected finish time.

### Falling and fading guardrails

A falling pin that remains falling too long is converted to a faded/removed state.

A fading pin that does not remove normally is removed after a grace period.

### Ball rolling guardrails

The rolling ball now has a maximum roll time. Invalid ball paths also resolve safely instead of leaving the game stuck in `rolling`.

### Resolving guardrails

The resolving phase now has a maximum duration. If it runs too long, it either:

- begins the reward/next-level sequence if all upright pins are gone, or
- returns to `playing` if upright pins remain.

## What did not change

- No new animations.
- No animation art changes.
- No pin count changes.
- No new gameplay rules.
- No intended change to normal pin, ball, or level behavior.
