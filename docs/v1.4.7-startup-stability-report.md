# Meowmoon Bowling v1.4.7 Startup Stability Fix

## Purpose

This release fixes an intermittent startup problem where the game could load without the opening "Tap anywhere to play" prompt and then not start until the browser was refreshed.

## What changed

- Added `ensureStartupReady(reason)`.
- Startup now checks that the first level, first message, pins, and opening phase are valid.
- Added a delayed startup readiness check shortly after load.
- `startLevel()` now uses the phase helper instead of assigning `game.phase` directly.
- On the title screen, pointer/tap handling now treats the title as true "tap anywhere" before checking for the cat long-press pause target.

## Why this is conservative

The startup readiness function only rebuilds the opening state if the first level is incomplete. It does not interfere with later levels or reward transitions.

## What did not change

- No new animations.
- No animation art changes.
- No gameplay rule changes.
- No pin count changes.
- No special animation frequency changes.
